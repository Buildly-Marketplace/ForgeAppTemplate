# Marketplace Manifest (BUILDLY.yaml)

This document describes the `BUILDLY.yaml` manifest file required for Buildly Marketplace submissions.

## Overview

Every Marketplace submission must include a `BUILDLY.yaml` file at the repository root. This file:

1. Provides metadata for the Marketplace listing
2. Defines test commands for validation
3. Specifies required UI hooks for E2E tests
4. Declares screenshot requirements

## File Location

```
your-repo/
├── BUILDLY.yaml    # Required - must be at repo root
├── marketplace/
│   └── screenshots/
│       ├── 01-home.png
│       └── 02-primary-action.png
└── ...
```

## Required Fields

### Identity

```yaml
# Unique identifier (lowercase, hyphens allowed)
id: my-app-template

# Display name for Marketplace
name: My App Template

# Short description (max 160 characters)
description: A brief description of what this template does

# Semantic version
version: 1.0.0
```

### Categorization

```yaml
# Template category
category: starter  # or: integration, component, full-stack

# Discoverable tags
tags:
  - python
  - api
  - dashboard
```

### Author

```yaml
author:
  name: Your Name
  email: you@example.com
  url: https://yoursite.com  # optional
```

### Legal

```yaml
# SPDX license identifier
license: MIT

# Repository URL
repository: https://github.com/org/repo
```

## Test Commands

Define commands that Marketplace CI will run to validate your submission:

```yaml
tests:
  # Docker-first (required for CI)
  docker:
    e2e: ./scripts/test-e2e-docker.sh
    screenshots: ./scripts/screenshots-docker.sh
  
  # Python-first (optional, for local dev)
  python:
    e2e: ./scripts/test-e2e-python.sh
    screenshots: ./scripts/screenshots-python.sh
```

### Requirements

- Docker test commands are **required**
- Commands must exit 0 on success, non-zero on failure
- No npm or Node.js dependencies allowed

## App Control

Define the single entrypoint for app control:

```yaml
app:
  control: ./ops/startup.sh
  modes:
    - python
    - docker
  commands:
    - start
    - stop
    - restart
    - status
```

## Screenshots

Marketplace listings display screenshots generated by your tests:

```yaml
screenshots:
  directory: marketplace/screenshots
  required:
    - 01-home.png
    - 02-primary-action.png
  generator: ./scripts/screenshots-docker.sh
```

### Screenshot Rules

1. **Naming**: Use numbered prefixes (`01-`, `02-`) for ordering
2. **Format**: PNG required, consistent dimensions (1280x800 recommended)
3. **Content**: Show app in different states
4. **Generation**: Must be auto-generated by Robot tests (not manually created)

### Why Auto-Generated?

- **Accuracy**: Screenshots always match actual app behavior
- **Freshness**: Updated automatically when UI changes
- **Validation**: Proves the app actually works as shown

## UI Hooks

Required `data-testid` attributes for E2E tests:

```yaml
ui_hooks:
  required:
    - app-root          # Main container
    - primary-action    # Primary CTA
    - panel             # Revealed content
```

These hooks ensure tests can reliably find elements regardless of styling changes.

### Adding Hooks to HTML

```html
<div data-testid="app-root">
  <button data-testid="primary-action">Click Me</button>
  <div data-testid="panel">Panel Content</div>
</div>
```

## Validation Rules

```yaml
validation:
  tests_must_pass: true       # All E2E tests must pass
  screenshots_required: true  # Required screenshots must exist
  manifest_required: true     # BUILDLY.yaml must be valid
  no_npm: true                # No npm/node dependencies
```

## Complete Example

```yaml
id: forge-app-template
name: ForgeAppTemplate
description: A minimal starter template for Buildly Marketplace
version: 1.0.0
category: starter

tags:
  - template
  - starter
  - python

author:
  name: Buildly Marketplace
  email: marketplace@buildly.io

license: MIT
repository: https://github.com/Buildly-Marketplace/ForgeAppTemplate

requirements:
  python: ">=3.10"
  docker: ">=20.10"

tests:
  docker:
    e2e: ./scripts/test-e2e-docker.sh
    screenshots: ./scripts/screenshots-docker.sh
  python:
    e2e: ./scripts/test-e2e-python.sh
    screenshots: ./scripts/screenshots-python.sh

app:
  control: ./ops/startup.sh
  modes: [python, docker]
  commands: [start, stop, restart, status]

screenshots:
  directory: marketplace/screenshots
  required:
    - 01-home.png
    - 02-primary-action.png
  generator: ./scripts/screenshots-docker.sh

ui_hooks:
  required:
    - app-root
    - primary-action
    - panel

validation:
  tests_must_pass: true
  screenshots_required: true
  manifest_required: true
  no_npm: true
```

## Validation

Before submitting, verify your manifest:

```bash
# Run tests
./scripts/test-e2e-docker.sh

# Generate screenshots
./scripts/screenshots-docker.sh

# Check screenshots exist
ls -la marketplace/screenshots/
```

## Common Issues

### "Missing required screenshot"

Run the screenshot generator:
```bash
./scripts/screenshots-docker.sh
```

### "Test command failed"

Check the test output in `artifacts/robot/`:
- `smoke-report.html` - Human-readable report
- `smoke-log.html` - Detailed execution log

### "Invalid BUILDLY.yaml"

Validate YAML syntax:
```bash
python3 -c "import yaml; yaml.safe_load(open('BUILDLY.yaml'))"
```

### "npm dependency detected"

Remove any `package.json`, `node_modules`, or `npm` references. Use `robotframework-browser-batteries` for Playwright without npm.
